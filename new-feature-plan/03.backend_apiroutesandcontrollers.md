Create all backend API routes and controllers for the Course Plan Automation feature.
This adds to the existing Express LMS backend.

CREATE THESE FILES:

1. server/src/routes/coursePlan.js
   Mount at: /api/faculty/course-plan  (protected: role=staff)
   Mount at: /api/hod/course-plan       (protected: role=hod)
   Mount at: /api/student/materials     (protected: role=student)

2. server/src/controllers/coursePlanController.js

IMPLEMENT THESE ENDPOINTS FULLY:

--- FACULTY ENDPOINTS ---

POST /api/faculty/course-plan/upload
  - Accept multipart/form-data with fields: file (docx), subjectId
  - Validate: faculty must be assigned to this subject
  - Save file to /uploads/course-plans/
  - Call coursePlanParser.js to extract data
  - If parse fails: return 400 with clear error message
  - Save to course_plan_uploads table
  - Save all topics to course_plan_topics table with display_order
  - Deactivate any previous upload for same faculty+subject
  - Return: { uploadId, totalTopics, units: [{unitName, topicCount}] }

GET /api/faculty/course-plan/:subjectId
  - Get active course plan for this faculty + subject
  - Return upload info + all topics grouped by unit
  - Include: completion stats per unit, overall completion %
  - Include: materials count per topic
  - Return 404 if no plan uploaded yet

GET /api/faculty/course-plan/topic/:topicId
  - Get single topic full details
  - Include: all pre-linked materials
  - Include: submitted report (if completed)
  - Validate: topic belongs to requesting faculty

POST /api/faculty/course-plan/topic/:topicId/report
  - Accept multipart/form-data: whatWasCovered (text), proofImage (file)
  - Validate: topic must be pending, must belong to this faculty
  - Save proof image to /uploads/proof-images/
  - Insert into topic_hour_reports
  - Update topic status to completed
  - Find all materials for this topic WHERE is_released=false
  - UPDATE all to is_released=true, released_at=NOW()
  - Find all students enrolled in this subject
  - For each student + each material: insert student_material_notifications
  - Queue emails: send async using nodemailer
    Email subject: New Study Material Available — [topic_name]
    Email body: Dear [student_name], [faculty_name] has completed the topic
    [topic_name] and released [N] study material(s). Login to view them.
  - Return success with: materialsReleased count, studentsNotified count

POST /api/faculty/course-plan/topic/:topicId/materials
  - Accept multipart/form-data: title, description, materialType, file OR url
  - Save file to /uploads/topic-materials/ if file upload
  - Insert into topic_materials with is_released=false, added_when='pre_linked'
  - Return created material

DELETE /api/faculty/course-plan/material/:materialId
  - Only allow if is_released = false
  - If released: return 400 'Cannot delete released material'
  - Delete file from disk + delete DB record

--- HOD ENDPOINTS ---

GET /api/hod/course-plan/reports
  - Query params: facultyId, subjectId, unitName, startDate, endDate, status
  - Return paginated list of topic_hour_reports
  - Include: faculty name, subject name, topic details, proof image URL
  - Include: materials released list per report
  - Default: last 30 days, all faculty, all subjects

GET /api/hod/course-plan/summary
  - Return dept-wide stats:
    totalPlannedHours, totalCompletedHours, totalPendingHours
    completionPercentage, activeSubjects, activeFaculty
  - Per-subject breakdown: subjectName, facultyName, planned, completed, %

--- STUDENT ENDPOINTS ---

GET /api/student/materials/:subjectId
  - Return all RELEASED topic_materials for subjects student is enrolled in
  - Group by topic name
  - Include: is_new flag (released within last 24 hours)

GET /api/student/notifications/materials
  - Return unread student_material_notifications for this student
  - Used for notification badge count

PUT /api/student/notifications/:notificationId/read
  - Mark notification as read
