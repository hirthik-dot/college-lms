Create the Course Plan Reports page for the HOD portal.
File: client/src/pages/hod/CoursePlanReports.jsx

DESIGN: Apple iOS style. Data-dense but clean.

DATA SOURCES:
- GET /api/hod/course-plan/summary → for stat cards
- GET /api/hod/course-plan/reports → for the main table (with filter params)

PAGE LAYOUT:

PAGE TITLE: 'Course Plan Reports' with subtitle 'Track faculty teaching progress'

STAT CARDS ROW (4 cards):
- Total Hours Planned (blue, calendar icon)
- Hours Completed (green, check icon)
- Hours Pending (orange, clock icon)
- Dept Completion % (blue, bar chart icon — shows as large % number)

SUBJECT PROGRESS PANEL (collapsible, below stat cards):
Title: 'Subject-wise Progress' with toggle button
When expanded: list of subjects, each showing:
- Subject name + faculty name
- Mini progress bar (filled = completed/planned)
- X/Y hours text + percentage
- Color: green if >75%, orange if 40-75%, red if <40%

FILTER BAR (card with filters):
- Faculty dropdown (All Faculty or specific name)
- Subject dropdown (All Subjects or specific)
- Unit dropdown (All Units, UNIT I, UNIT II, UNIT III)
- Date range: From date + To date pickers
- Status: All / Completed only
- Two buttons: 'Apply Filters' (blue) + 'Clear' (ghost)
- 'Export to Excel' button on the right side

RESULTS TABLE:
Show total count above table: 'Showing X results'

TABLE COLUMNS:
Faculty | Subject | Unit | Topic | Periods | Submitted At | What Was Covered | Proof | Materials

- Faculty: Avatar (24px) + name
- Subject: Subject code badge + name
- Unit: Small colored badge (Unit I = blue, Unit II = purple, Unit III = green)
- Topic: Topic name (truncated to 40 chars with tooltip for full name)
- Periods: Number in grey pill
- Submitted At: Formatted date + time (e.g. 'Today 2:30 PM' or 'Mar 15, 3:00 PM')
- What Was Covered: First 60 chars + '...' — click row to see full
- Proof: Small image thumbnail (40x40px) — click opens full size in modal
- Materials: '3 files' badge in green — click shows list of file names

ROW CLICK — Expanded Detail Panel:
Clicking a row expands a detail panel below it (not a separate page):
- Full text of what was covered
- Full-size proof image (max 400px wide)
- List of all materials released: icon + title + type badge + released timestamp
- Topic details: unit, CO/Bloom, teaching method, reference book

EMPTY STATE (when no reports match filters):
- Icon + 'No reports found for the selected filters'
- 'Clear Filters' button

EXPORT TO EXCEL:
On click: use SheetJS to generate Excel file with all filtered data.
Columns: Faculty, Subject, Unit, Topic, Periods, Submitted At,
What Was Covered, Materials Released Count, Proof Image URL

Use React Query with filter params as query keys (refetch when filters change).
Pagination: 20 rows per page with prev/next buttons.
Generate complete file — fully working, production ready.
