Create a Word document parser for the college LMS Course Plan feature.
File: server/src/utils/coursePlanParser.js

This parser receives a .docx file buffer and extracts the Course Plan table.

USE: mammoth library (already in package.json) for initial extraction,
then use docx library (npm: docx-extractor or officegen) for table parsing.
Actually use: npm package 'docx' with JSZip to directly read the XML
OR use: 'mammoth' with raw XML extraction.
Best approach: use the 'docx' npm package to read the file as XML and
parse the w:tbl elements directly to get table structure.

THE DOCUMENT STRUCTURE TO PARSE:
The Word document contains multiple tables. We want TABLE 2 specifically.
Identify it by: the first row containing 'Course Plan' spanning all columns.

WITHIN THE COURSE PLAN TABLE, detect these row types:
1. Title row: first cell text contains 'Course Plan' → skip
2. Subject row: first cell contains 'Sub Code & Name' → extract subject info
3. Unit header row: first cell contains 'UNIT' AND another cell contains 'Target Hours'
   → extract unit name and target hours number
4. Column header row: first cell text is 'S. No' or 'S.No' → skip
5. Topic data row: first cell is a number (1,2,3...) → extract all 6 fields
6. Total hours row: any cell contains 'Total Hours' → skip

FOR TOPIC DATA ROWS, extract these columns in order:
- Column 0: sno (parse as integer)
- Column 1: periods_required (parse as integer, default 1)
- Column 2 or 3: topic_name (take the longer text if columns are merged)
- Column 4 or 3: reference_book
- Column 5 or 4: teaching_method
- Column 6 or 5: co_bloom

NOTE: Some rows have merged cells — handle this gracefully.
If a cell is empty due to merge, use the previous non-empty value.

ALSO EXTRACT from TABLE 0 (the first table — course info):
- academic_year: row where first cell = 'Academic year', take 3rd cell value
- subject_code_and_name: row where first cell = 'Course code and title', take 3rd cell
- contact_hours: row where first cell = 'Contact Hours', take 3rd cell
- type_of_course: row where first cell = 'Type of Course', take 3rd cell

RETURN this structured object:
{
  academicYear: string,
  subjectCode: string,
  subjectName: string,
  totalTheoryHours: number,
  totalLabHours: number,
  units: [
    {
      unitName: string,
      targetHours: number,
      topics: [
        {
          sno: number,
          periodsRequired: number,
          topicName: string,
          referenceBook: string,
          teachingMethod: string,
          coBloom: string,
          displayOrder: number  // global sequential number across all units
        }
      ]
    }
  ],
  totalTopics: number,
  parseErrors: []  // any warnings during parsing
}

Write comprehensive error handling.
If Course Plan table not found: throw new Error('COURSE_PLAN_NOT_FOUND')
If less than 5 topics found: throw new Error('INSUFFICIENT_TOPICS')

Also write a test: server/src/utils/coursePlanParser.test.js
that tests the parser with a mock document structure.

Generate the complete file — fully working, production ready.
