Create the Submit Hour Report page for the Faculty portal.
File: client/src/pages/staff/SubmitHourReport.jsx

This page opens when faculty clicks 'Submit Report' on a pending topic card.
It receives topicId as a URL parameter.
Route: /staff/course-plan/submit/:topicId

DESIGN: Apple iOS style. Clean, focused form. No distractions.

STEP 1 — Load topic details:
GET /api/faculty/course-plan/topic/:topicId
Show LoadingSkeleton while loading.
If topic is already completed: redirect to CoursePlanManager with toast
'This topic has already been submitted.'

PAGE LAYOUT (single column, max-width 680px, centred):

BACK BUTTON: '← Back to Course Plan' at top left

TOPIC DETAILS CARD (read-only, light blue background):
- Label: 'Topic to Submit'
- Unit name (small grey text)
- Topic name (large bold text)
- Three pills in a row: Periods badge, CO/Bloom badge, Reference badge
- Teaching method (grey text with method icon)

FORM SECTION:

Field 1 — What did you cover?
- Label: 'What did you cover in this session? *'
- Hint text: 'Describe what you taught, any extra topics covered, student
  activities, or notes about this session.'
- Large textarea, min height 120px, max height 400px (auto-grows)
- Character counter: '0 / minimum 20 characters'
- iOS-style: no harsh border, subtle grey background, focus = blue ring

Field 2 — Upload Proof
- Label: 'Upload Proof of Teaching *'
- Hint text: 'Photo of board/screen, screenshot of presentation, or student activity photo'
- Drag-and-drop zone: dashed border, upload cloud icon
- Text: 'Drag photo here or click to browse'
- Accepted: JPG, PNG, HEIC — Max 10MB
- After selection: show image preview (100% width, max 200px height, object-fit: cover)
- Show file name and size below preview
- X button to remove and re-select

Field 3 — Study Materials to Release
- Label: 'Study Materials to Release to Students'
- Hint: 'These will become visible to all enrolled students when you submit.'

SUB-SECTION A: Pre-linked materials:
- List of materials already linked to this topic
- Each: checkbox (checked by default) + file type icon + title + file size
- If no pre-linked materials: show subtle text 'No materials pre-linked'

SUB-SECTION B: Add more now:
- 'Add Material' button — opens inline upload form below
- Inline form: Title input, Type dropdown, File upload OR URL input, Add button
- Added materials appear in the list above with a 'NEW' badge

SUBMIT BUTTON:
- Full-width iOS blue button: 'Submit Hour Report'
- Disabled until: whatWasCovered >= 20 chars AND proofImage selected
- When disabled: grey, not clickable
- When enabled: iOS blue, hover darkens slightly

ON SUBMIT CLICK — show confirmation modal:
- Title: 'Confirm Submission'
- Body: 'This will release [N] study materials to [M] enrolled students.
  This action cannot be undone.'
- Two buttons: 'Cancel' (ghost) and 'Confirm & Submit' (iOS blue)

ON CONFIRM:
- POST /api/faculty/course-plan/topic/:topicId/report
  FormData with: whatWasCovered, proofImage, materialIds (pre-linked checked ones),
  newMaterials (newly added files/URLs)
- Show full-page loading overlay: spinning circle + 'Submitting report and releasing materials...'
- On success: navigate to /staff/course-plan with success toast:
  '✓ Hour submitted. [N] materials released to [M] students.'
- On error: dismiss loading, show error toast with retry option

Generate complete file — fully working, production ready.
