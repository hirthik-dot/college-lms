DESIGN: Apple iOS style — white cards, soft shadows, rounded corners,

- Dropdown showing all subjects assigned to this faculty
- Each option: Subject Name (Subject Code) — e.g. Python for Data Science (22AD402)
- When a subject is selected, load its course plan via GET /api/faculty/course-plan/:subjectId

- Illustration (use a document icon from lucide-react)
- Text: 'No Course Plan uploaded for this subject yet'

- On file select: POST /api/faculty/course-plan/upload with subjectId
- Show loading state: animated spinner + 'Scanning document...' text
- On success: show toast 'Course Plan uploaded — X topics found' then reload
- On error COURSE_PLAN_NOT_FOUND: show error 'Course Plan table not found.

IF PLAN EXISTS — show full page:

PROGRESS SECTION (card at top):
- Course name and code (from the uploaded document)
- Academic year badge
- Three stat pills: X Completed (green), Y Pending (grey), Z Total
- A horizontal progress bar: filled portion = completed/total * 100%
- Bar color: green if >75%, orange if 50-75%, blue if <50%
- 'Replace Document' button (small, secondary style) — allows re-upload

UNIT SECTIONS:
- One accordion/collapsible section per unit

- Default state: all units expanded
- Inside each unit: grid of topic cards (1 column on mobile, 2 on desktop)

TOPIC CARD (for each topic in a unit):

- Top row: S.No badge (grey pill) + CO/Bloom badge (coloured by CO level) + Status badge

  Left: Materials count (paperclip icon + '2 materials')

ADD MATERIALS PANEL (slides down below card when 'Add Materials' clicked):
- File upload area: drag-drop zone + 'or click to browse'
- URL input option (for video links)
- Title input (required)
- Material type selector (PDF, PPT, DOC, Video, Link)
- Upload button
- List of already-linked materials below with delete option

VIEW REPORT PANEL (slides down for completed topics):
- Grey background section showing:
